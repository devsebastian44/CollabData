# Contributing to CollabData

First off, thank you for considering contributing to CollabData! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## How Can I Contribute?

### Reporting Bugs
- Check the [Issues tab](https://github.com/devsebastian44/CollabData/issues) to see if the bug has already been reported.
- If not, open a new issue with a clear title and description.

### Suggesting Enhancements
- Open an issue with the tag `enhancement`.
- Describe the feature you'd like to see and why it would be useful.

### Pull Requests
1. **Fork** the repository.
2. Create a **feature branch** (`git checkout -b feature/amazing-feature`).
3. **Commit** your changes using [Conventional Commits](https://www.conventionalcommits.org/).
4. **Push** to the branch (`git push origin feature/amazing-feature`).
5. Open a **Pull Request** against the `main` branch.

## Local Development
1. Clone your fork.
2. Run `npm install`.
3. Copy `.env.example` to `.env.local` and fill in your credentials.
4. Run `npm run dev`.

## Testing
Please ensure that your changes pass all tests:
```bash
npm run test:run
npm run lint
npm run typecheck
```

## Professional Standards
- Maintain clean, modular code.
- Ensure all new features are documented.
- Add tests for new logic.
